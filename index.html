Class and ID Selectors

In this exercise, I assigned class and ID attributes to various elements and then applied CSS rules to style them. The key aspects were:

  • Deciding whether to use class or ID attributes for each element, based on their styling needs.
  • Styling elements using non-keyword color values (like RGB, HEX, or HSL) and specific font properties.
  • Focusing on selector syntax in CSS to apply styles correctly to the targeted elements.
This exercise was aimed at understanding practical application of class and ID selectors in CSS.

Grouping Selectors

This exercise builds off of the previous challenge. Instead of having a single element with two different rules applied, I instead gave two elements each a unique class name, then added rules for styles that both elements shared as well as their own unique styles.

Chaining Selectors

The goal of this exercise was to enhance my skills in chaining different CSS selectors. I was given HTML code and two images, each assigned two class names, where one class name was shared between both images. My task was to apply unique styles to each image using chained selectors, despite the shared class.

Descendant Combinator

Cascade

The final exercise was designed to deepen my understanding of the CSS cascade, focusing on specificity and rule order. Instead of creating new CSS rules, my task was to edit an existing CSS file that accompanied a fully written HTML file with the following constraints:

  • I had to identify and resolve issues related to specificity and rule order in the provided CSS file. This meant analyzing how different CSS rules are applied to HTML elements and determining which rules take precedence.
  • I could only edit the CSS file by modifying selectors, adjusting the order of declaration blocks, or adding and removing selectors within a declaration block.
  • I couldn't alter the HTML code or the actual styles (properties and values) defined in the CSS.
